System Overview

The Walker Tournament API is the central nervous system for tournament management. It orchestrates the lifecycle of combat events, from initial registration to the final championship match.

Mission Objective

To provide a robust, secure, and real-time backend for managing high-stakes Walker Bracket Tournaments. The system ensures data integrity via optimistic concurrency and delivers automated bracket progression.

Prerequisites

  • .NET 9.0 SDK or later
  • Node.js 18+ and npm
  • Git for version control
  • SQLite (included with .NET)

Getting Started

Backend Setup

  1. Navigate to the backend directory 
    cd backend/src/WalkerTournament.Api
  2. Configure JWT Secret 
    dotnet user-secrets init
dotnet user-secrets set "Jwt:SecretKey" "your-super-secret-key-minimum-32-characters-long"
  3. Apply Database Migrations 
    dotnet ef database update
  4. Run the Backend 
    dotnet run

    The API will be available at https://localhost:5001 (or http://localhost:5000)

Frontend Setup

  1. Navigate to the frontend directory 
    cd frontend
  2. Install Dependencies 
    npm install
  3. Configure API Endpoint

    Edit src/lib/api.js to set the correct backend URL:

    const API_BASE_URL = 'https://localhost:5001';
  4. Run the Development Server 
    npm run dev

    The frontend will be available at http://localhost:5173

Configuration Files

Backend: appsettings.json

{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=walkertournament.db"
  },
  "Jwt": {
    "Issuer": "WalkerTournamentAPI",
    "Audience": "WalkerTournamentClients",
    "ExpiryMinutes": 60
  }
}
Important: The Jwt:SecretKey is stored in user secrets for security, not in appsettings.json.

Frontend: src/lib/api.js

Update the API_BASE_URL constant to match your backend URL:

  • Development: https://localhost:5001
  • Production: Your deployed API URL

Core Workflows

1. The Tournament Loop

  • Draft: Admin creates a tournament. Configures XP rewards and deadlines.
  • Registration: Public walkers join via the open endpoint.
  • Lock & Draw: Admin locks the tournament (closing registration) and triggers the bracket generation engine.
  • Combat: Admin resolves matches by declaring winners. The system automatically promotes winners to the next round.
  • Victory: The final match determines the champion, awarding XP and closing the event.
